# roomdoo-locks-tesa

TESA Smartair provider for Roomdoo smart lock integrations.

Implements the `BaseLockProvider` contract (`grant_access` / `modify_access` /
`revoke_access` / `test_connection`) on top of the TESA Smartair SOAP API
(`GuestsWebService`, `UsersWebService`, `DoorsWebService`) using `zeep`.

## Installation

From the root of the monorepo:

    pip install -e "./roomdoo-locks-base"
    pip install -e "./roomdoo-locks-tesa"

## Usage

    from datetime import datetime, timezone, timedelta
    from roomdoo_locks_tesa import TesaSmartairProvider

    provider = TesaSmartairProvider(
        host              = "192.168.1.50",   # hostname or IP, no scheme
        operator_name     = "YOUR_OPERATOR",
        operator_password = "YOUR_PASSWORD",
        port              = 8181,             # default
        verify_ssl        = False,            # default
    )

    starts_at = datetime.now(timezone.utc)
    ends_at   = starts_at + timedelta(days=2)

    # Grant one guest access to a set of rooms for a validity window.
    grant = provider.grant_access([81, 82], starts_at, ends_at)
    print(grant.pin)         # 4-digit PIN the guest uses on every room
    print(grant.ref)         # opaque handle: store it verbatim
    print(grant.starts_at, grant.ends_at)

    # Extend / shorten the window (only the expiration date is changed).
    grant = provider.modify_access(grant.ref, starts_at, ends_at + timedelta(days=1))

    # Revoke (idempotent): checks out / cancels the stay on every room.
    provider.revoke_access(grant.ref)

## Constructor parameters

| Parameter         | Type | Required | Default | Description                                   |
|-------------------|------|----------|---------|-----------------------------------------------|
| host              | str  | Yes      |         | Smartair server hostname or IP (no scheme)    |
| operator_name     | str  | Yes      |         | API operator login                            |
| operator_password | str  | Yes      |         | API operator password                         |
| port              | int  | No       | 8181    | Server port                                   |
| verify_ssl        | bool | No       | False   | Require a valid server certificate            |

## Notes

- **Authentication** is stateless: `operatorName` + `operatorPassword` are sent
  with every SOAP request. There is no session token.
- **Check-in vs pre-check-in.** A `checkin` only applies when the guest is
  entering *now* (the room becomes occupied immediately). When `starts_at` is in
  the future the whole grant is created through `precheckin` instead, which
  returns a `preAssignationId`. The choice is decided from `starts_at` against
  the current UTC time and recorded in the `ref`.
- **PIN.** Generated by the provider unless one is passed to `grant_access`. The
  first digit is always non-zero (TESA does not allow a leading-zero PIN); the
  rest are `0-9`. A 4-digit PIN is used by default.
- **PIN collisions.** Smartair rejects a PIN already active on another lock with
  an overlapping window. An auto-generated PIN is retried with a fresh value (up
  to `_MAX_PIN_ATTEMPTS`); a caller-supplied PIN raises `LockPinCollisionError`.
- **One PIN across rooms.** A grant pushes the *same* PIN to every room of the
  set. If any room fails, the whole grant is rolled back (all-or-nothing).
- **`ref` is opaque**: it carries `precheckin` plus the per-room handles
  (`roomId` / `preAssignationId`) needed by `modify_access` / `revoke_access`.
  Store it verbatim and never parse it.
- **modify_access** only changes the expiration date — TESA does not allow
  modifying the PIN or the start date. The PIN is unchanged; the caller keeps
  the one returned by `grant_access`.
- Dates must be timezone-aware UTC `datetime` objects.

## Extras (beyond the contract)

- `find_all_rooms()` / `find_all_occupied_rooms()` / `get_room_info(room_id)` —
  room state listing.
- `modify_grants(lock_id, grants)` — replace the grant list of an active checkin.
- `add_pin_user(...)` / `delete_user(username)` — permanent staff PIN users.
- `open_door(door_id)` — remotely open a wireless door.
